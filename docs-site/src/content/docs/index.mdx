## Why html-to-markdown

<CardGrid>
	<Card title="Rust core" icon="setting">
		Single-pass DOM walk written in Rust. The same code path runs from Python, the browser, and the
		CLI — no per-language conversion logic.
	</Card>
	<Card title="16 language surfaces" icon="translate">
		Rust plus generated packages for Python, TypeScript, Go, Ruby, PHP, Java, C#, Elixir, R, C,
		WebAssembly, Swift, Dart, Kotlin Android, and Zig.
	</Card>
	<Card title="Three output formats" icon="document">
		Markdown (CommonMark) by default, plus Djot and plain text via `output_format`. The same options
		apply to every format.
	</Card>
	<Card title="Metadata extraction" icon="magnifier">
		Document title, Open Graph, Twitter Card, JSON-LD, links, and images in one pass. Enabled by
		default — disable with `extract_metadata: false`.
	</Card>
	<Card title="Table extraction" icon="random">
		HTML tables into `result.tables` with structured cells, row/column spans, and header flags,
		alongside the rendered Markdown.
	</Card>
	<Card title="Visitor pattern" icon="laptop">
		40 callbacks on the `HtmlVisitor` trait to skip, replace, or preserve nodes. Kotlin Android
		excludes visitor support. Zero cost when unused.
	</Card>
</CardGrid>

## Quick example

<Tabs syncKey="lang">
<TabItem label="Python">

```python title="main.py"
from html_to_markdown import convert

result = convert("<h1>Hello</h1><p>World</p>")
print(result.content)
print(result.metadata)
```

</TabItem>
<TabItem label="TypeScript">

```typescript title="index.ts"
import { convert } from "@xberg-io/html-to-markdown";

const result = convert("<h1>Hello</h1><p>World</p>");
console.log(result.content);
console.log(result.metadata);
```

</TabItem>
<TabItem label="Rust">

```rust title="src/main.rs"
use html_to_markdown_rs::convert;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let result = convert("<h1>Hello</h1><p>World</p>", None)?;
    println!("{}", result.content.unwrap_or_default());
    Ok(())
}
```

</TabItem>
</Tabs>
